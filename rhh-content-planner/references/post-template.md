## Before You Write — Pre-Draft Checklist

Complete all four before touching the hook.

```
[ ] PILLAR
    Which pillar does this post belong to?
    → The Stakes / In Practice / Mental & Digital Models
    If it fits two, name the primary and state why the other was set aside.

[ ] POST TYPE
    → Diagnostic / Contrarian / Build Log / Case Narrative / Model / Founder Note

[ ] READER STATE
    Who is reading this — and where are they right now?
    Name the specific emotional state, not just the avatar label.
    
    CREDPRO examples:
    → "Privately aware they're spending time on work that doesn't need them.
       Hasn't quantified it. Hasn't named it."
    → "Has tried ChatGPT. Didn't trust it with anything real. 
       Now skeptical of everything in the category."
    → "Three years in. Workflow is working. Starting to wonder 
       if they've already fallen behind."
    
    EXEC-DEV examples:
    → "Has tried delegation frameworks. They didn't stick. 
       Intellectually unconvinced another framework solves this."
    → "Aware the gap exists. Privately frustrated no one has 
       named it without pitching something."

    Write one sentence. This does not appear in the post.
    It is your constraint for everything that follows.

[ ] TRIGGER MOMENT
    What near-miss or borrowed pain event surfaces this post?
    The reader finding organic content is in consideration mode —
    not in crisis. They're seeking clarity before commitment.
    Reactive readers don't read Substack posts. They call a lawyer.
    
    Name the specific trigger:
    → A colleague's situation that made the gap visible
    → A task on their desk they almost delegated and didn't
    → A tool they tried that broke their trust for the category
    → A growth moment where the infrastructure didn't keep pace
    
    If the trigger is a full crisis, the reader isn't here.
    Sharpen until it's a near-miss.
```

## Short Post Template — 400 words

**When to use:** The Stakes and Mental & Digital Models primarily.
One idea. One cut. Lands and leaves.

---

### HOOK
```
~50 words / 2–3 sentences

Name the cost or contradiction.
No setup. No questions. No warmup.
Land directly on the thing they've been living.
Reader thinks: "that's my week."

FAIL: If sentence three is still setting up,
cut sentences one and two and start there.
```

---

### TENSION
```
~80 words / 1 paragraph

One specific scenario.
Something they've tried, lived, or avoided.
Tied to the near-miss or borrowed pain trigger named in pre-draft.
No framework. No solution. Not yet.

FAIL: If this could apply to any professional
in any context, it's too generic.
Sharpen the scenario or change the angle.

FAIL: If this describes a full crisis,
the reader isn't here. Pull back to the near-miss moment.
```

---

### TURN
```
~30 words / 1–2 sentences

The reframe.
Not the answer — the right question,
or the thing they haven't named yet.
Earned from TENSION.

FAIL: If it needs three sentences,
the tension didn't earn it.
```

---

### MODEL
```
~150 words / 1–2 paragraphs

The portable insight.
Grounded in the situation from TENSION —
never abstracted away from it.

Pillar guidance:
→ The Stakes: name the shift and its specific cost
→ In Practice: what was built / what failed
→ Mental & Digital Models: the portable thinking framework

Capability test (replaces transfer test):
Does the reader finish this post permanently more capable
of making this category of decision — not just this instance of it?
If the model only applies once, it's a tip. Rewrite it as a principle.

FAIL: If the reader needs to return to the author to use this,
the model isn't finished.
```

---

### CLOSE
```
~40 words / 1–2 sentences

One clean implication. No summary. No inflation.
Weight or possibility — not both.

FAIL: If it restates the MODEL,
delete it and use the last MODEL sentence as the close.
```

---

### CTA
```
1 line. Final. Not repeated.
Appears after the close. Earned, not announced.

→ "If you want to see where your gap is:
   Assess Your Delegation Gap — 5 minutes, free."
```

## Long Form Template — 1,000+ words

**When to use:** In Practice primarily. Also Mental & Digital Models
when the framework requires a real professional situation to land.
Length comes from evidence, not elaboration.

---

### HOOK
```
~60 words / 2–3 sentences

Same rule as short.
Cost or contradiction. No warmup.

FAIL: If it requires the next section
to make sense, rewrite it.
If it could open a short post, it's right.
```

---

### TENSION
```
~200 words / 2–3 paragraphs

The full situation.
What was tried. What failed.
What it cost — in real terms, not abstractions.
Tied to the near-miss or borrowed pain trigger named in pre-draft.
Specific enough that the reader could verify or replicate
what's described.

FAIL: If it could apply to any professional
in any context, it's too generic.
Sharpen the scenario or change the angle.

FAIL: If this describes a full crisis,
the reader isn't here. Pull back to the near-miss moment.
```

---

### TURN
```
~80 words / 1 paragraph

The pivot from cost to mechanism.
Not the solution — the frame that makes the solution visible.
Stakes of the pivot are visible because TENSION earned them.

FAIL: If it introduces the solution,
it's doing MODEL's job. Pull back.
```

---

### MODEL
```
~400 words / 3–4 paragraphs

The insight, fully developed.
Grounded in the situation from TENSION —
never abstracted away from it.

Pillar guidance:
→ The Stakes: the shift named and evidenced
→ In Practice: the full build, including failure
→ Mental & Digital Models: framework grounded in a real
   professional situation, not abstracted from one

Capability test:
Does the reader finish this post permanently more capable
of making this category of decision — not just this instance of it?
If the model only applies once, it's a tip. Rewrite it as a principle.

FAIL: If length comes from elaboration rather than evidence,
cut to short format.

FAIL: If the reader needs to return to the author to use this,
the model isn't finished.
```

---

### IMPLICATION
```
~150 words / 1–2 paragraphs

*** Long form only. Not available in the short format. ***

What changes downstream if the model is true.
Not "therefore do this."
"Here's what this decision looks like differently
now that you have this model."

The question this section answers:
What can the reader now decide that they couldn't before?
Not a single instance — a category of decisions.

Earned by the evidence in MODEL.
If MODEL is thin, this section will sound like a pitch.
Fix MODEL before writing IMPLICATION.

FAIL: If it sounds like a CTA,
the MODEL didn't earn it. Rewrite MODEL first.

FAIL: If a short post draft has this section,
it's a long post that doesn't know it yet.
Remove IMPLICATION and tighten to short format.
```

---

### CLOSE
```
~60 words / 2–3 sentences

Same rule as short — no summary, no restatement.
One clean weight or possibility.
The post ends here, not at the CTA.

FAIL: If it restates the MODEL or IMPLICATION,
delete it and end on the last IMPLICATION sentence instead.
```

---

### CTA
```
1 line. Same line. Same placement. Earned. Final.

→ "If you want to see where your gap is:
   Assess Your Delegation Gap — 5 minutes, free."
```

## Voice Rules — Apply to Both Formats

```
DO:
→ Short sentences. Blunter than the brand documents suggest.
→ Name the cost before naming the fix.
→ Earned authority — the practitioner who lived the problem.
→ Specific scenarios over general observations.
→ Plain language throughout.

DO NOT:
→ AI consultant register. No productivity pitch.
→ Performance. No warmup. No rhetorical questions.
→ Assertions without demonstration.
→ Summarising in the close.
→ Repeating the CTA.
→ Any sentence that could appear on any other 
   professional services website.
```

## Quality Check — Run Before Finalising

```
Both formats:

[ ] Does the hook name cost or contradiction without pitching?
[ ] Is the tension tied to a near-miss or borrowed pain moment
    — not a full crisis?
[ ] Is the tension specific enough to verify or replicate?
[ ] Does the turn earn its pivot from the tension?
[ ] Does the model pass the capability test —
    permanently more capable, not just this instance?
[ ] Does the close land without summarising?
[ ] Does the CTA appear once, at the end, earned?
[ ] Is every sentence free of AI consultant register?
[ ] Could the reader use the model without returning to the author?

Long form only:

[ ] Does the implication answer "what can the reader now decide
    that they couldn't before?" — not just "what should they do?"
[ ] Is the implication earned by MODEL evidence,
    not asserted on top of a thin MODEL?
[ ] Does length come from evidence, not elaboration?
```

## Format Reference

```
Short post:  ~400 words  →  6 sections  →  no Implication
Long form:  ~1,000 words →  7 sections  →  Implication included

The only structural difference is the Implication section.
Everything else is the same skeleton at different evidence depth.

Pillar determines the angle, not the format.
Any pillar can run in either format.
Format is determined by how much evidence the model requires
to pass the capability test — not by topic importance or length preference.
```
